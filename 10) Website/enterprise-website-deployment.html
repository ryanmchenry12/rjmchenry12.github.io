Overview

What We Built

Ambercrombie & Phish is a simulated enterprise e-commerce environment built from the ground up on Proxmox. The project models a real company network a clothing store inspired by Abercrombie & Fitch complete with WAN/LAN/DMZ segmentation, centralized authentication, a full-stack web application, a backend database, and an internal mail system.

The goal was to mirror a production-grade infrastructure, satisfy CCPA and COPPA compliance requirements, and withstand a live penetration test by an opposing team. Every layer firewall rules, Active Directory policies, database access controls, and SIEM alerting was hardened before pen-test week.

Infrastructure

Network Overview

The network was redesigned from a flat topology into four isolated segments managed by OPNsense. Firewall rules enforce strict inter-VLAN traffic controls, and an ICMP block combined with workstation-to-workstation rules reduced lateral movement risk significantly. Click the diagram to enlarge.

Application Layer

The Web App

🛒
Storefront
Homepage, product detail pages, and a full order form system — styled after a mid-size clothing retailer selling men's, women's, and children's apparel.
🤖
AI Support Bot
Integrated AI chat widget for customer support, reducing manual ticket volume and providing 24/7 initial triage for common queries.
🔑
Auth & Accounts
Sign-in and user account features with session management. Designed to meet CCPA and COPPA compliance standards from day one.
📧
Contact & Support
Dedicated support page routing to the internal mail system, enabling realistic SMTP / IMAP traffic simulation across the enterprise network.
IIS & PHP on Win-Webserver
The web tier runs IIS with PHP on a Windows Server host inside the DMZ. All dynamic requests query the MySQL backend, which is accessible only from the web server — keeping the database fully off the public-facing segment.
Backend Services

Database & Mail

🗄️
MySQL Database
Stores all product catalog, order, and user data in structured relational tables. Queries are issued exclusively by the web server — no direct external access — to minimize the attack surface. Handles users, products, and orders in the backend with fast indexed retrieval.
✉️
Mail System via MailEnable
Running on a RHEL server with Postfix and Dovecot, the mail system handles SMTP and IMAP/POP3 traffic for the internal domain. Support and sales accounts generate realistic enterprise traffic patterns and integrate with the database for order notifications.

Compliance

CSET Assessment

The project was evaluated against the CSET (Cyber Security Evaluation Tool) compliance framework. Overall scores reflected strong security posture across protection and detection domains. The primary gap identified was in the recovery domain — the team lacked a formalized incident recovery plan, which would be a priority in a real production environment.

✅
Identify & Protect
Network segmentation, AD least-privilege, and firewall hardening all contributed to a solid score across these domains.
📊
Detect
Wazuh SIEM coverage across five endpoints provided robust detection capability. Zero critical alerts during pen-test week validated the ruleset.
⚠️
Recover
The team's main improvement area. A formalized recovery plan with documented runbooks and backup procedures would strengthen this domain in future iterations.
Takeaways

Lessons Learned

01
Firewall Rules Require Iteration Crafting effective floating rules that block lateral movement without breaking legitimate traffic took multiple refinement cycles. Documentation from the start would have saved time.
02
AD Least Privilege is Non-Trivial Scoping Active Directory permissions correctly — especially across DNS, IT Admin, and guest accounts — required careful testing to avoid over-permissioning.
03
Dashboards Drive Visibility Building Wazuh dashboards early on would have surfaced anomalies faster. Visibility tooling should be day-one infrastructure, not an afterthought.
04
AI Has Limits The AI support bot needed significant prompt tuning and edge-case handling. Treating AI output as a first draft rather than a final answer was the right call.
05
Track Progress Continuously A living Gantt chart and weekly check-ins kept the team aligned. Projects of this scope require structured progress tracking to avoid scope drift near deadlines.
